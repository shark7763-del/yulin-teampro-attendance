# 育林國中體育班出席管理系統 2.0 — 教練戰情室版

GitHub Pages 前端 + Google Apps Script API + Google Sheet。手機優先、深藍白行政風格。
設計原則：**手機優先、教練三秒看懂、助教不容易按錯、點完名馬上回報、系統自動提醒需要關心的學生**。

## 四大分頁
1. **🏠 戰情室**：自動顯示今日日期/星期、目前時段（早晨操／下午專長／晚晚自習）、今日六大統計（應到/出席/遲到/請假/未到/尚未點名）、需要關心學生、一鍵教練群＋家長群回報。
2. **✏️ 點名**：手機卡片式，每人四個大按鈕（✅出席 ⏰遲到 📝請假 ❌未到）＋顏色＋固定統計列＋篩選 chips（全部/尚未點名/出席/遲到/請假/未到）。請假可選原因、遲到可選分鐘、可加備註。
3. **📊 報表**：今日／本週／本月，全隊出席率＋每人紅黃綠燈，可匯出 Google Sheets 或複製 CSV。
4. **👥 學生**：名單管理、晚自習報名星期、個別出席紀錄。

## 一、建立試算表與 Apps Script 後端
1. Google 雲端硬碟 → 新增 → Google 試算表，命名「育林國中體育班出席管理系統」。
2. 擴充功能 → Apps Script。
3. 把 **Code.gs** 全部內容貼到 `Code.gs`。
4. 存檔。

## 二、初始化（只做一次）
- **全新安裝**：執行 `setup()`（建 4 個工作表＋預設帳號）→ 再執行 `importRoster()`（匯入 41 位學生）。
- **舊版升級到 2.0**：執行一次 `upgradeSchema()`。它會保留你既有資料，只補上新的 Attendance 欄位（星期/班級/請假原因/遲到分鐘/備註/建立時間/修改時間），舊的點名紀錄不會遺失。

## 三、部署 Apps Script API
1. 部署 → 新增部署作業 → 網頁應用程式。
2. 執行身分：**我**；誰可以存取：**任何人**。
3. 複製 `…/exec` 網址。GitHub 前端會用這個網址讀寫 Google Sheet。
> 改程式後要：部署 → 管理部署作業 → 編輯（鉛筆）→ 版本選「新版本」→ 部署。網址不變。

## 三之一、GitHub Pages 前端與 PWA 安裝
前端 App 放在 GitHub Pages，檔案包含 `index.html`、`manifest.json`、`service-worker.js`、`icons/`。

GitHub Pages 啟用方式：
1. GitHub repo → Settings → Pages。
2. Source 選 `Deploy from a branch`。
3. Branch 選 `main`，資料夾選 `/ (root)`。
4. 儲存後等待 GitHub 產生網址。

### Android / Chrome
1. 用 Chrome 開啟 GitHub Pages 網址。
2. 若畫面右下出現「安裝App」，直接點選。
3. 若沒有出現，點 Chrome 右上選單 →「加入主畫面」或「安裝應用程式」。

### iPhone / iPad
1. 用 Safari 開啟 GitHub Pages 網址。
2. 點分享按鈕 →「加入主畫面」。
3. 名稱可用「育林出席」。

> 不需要在 Apps Script 新增 `logo192` / `logo512` HTML 檔。

## 四、預設帳號（請至 Users 工作表自行修改 PIN）
登入採「選身分 + 4 位數 PIN」。

| 身分 | 帳號 | PIN | 可看專項 |
|------|------|-----|----------|
| 系統管理員 | admin | 9999 | 全部 |
| 田徑教練 | track | 1111 | 田徑 |
| 跆拳道教練 | tkd | 2222 | 跆拳道 |
| 武術教練 | wushu | 3333 | 武術 |

- 管理員：看所有專項、可新增/編輯/刪除學生、匯出報表。
- 教練：只看自己專項，點名＋回報＋看報表。
- 舊表密碼是英文（admin123 等）時，執行一次 `resetDefaultPins` 改成上表數字。
- 登入後手機記住身分，下次直接進系統；換人按右上「登出」。

## 五、Google Sheet 工作表欄位

### 1. Students（學生資料）
| 學生ID | 姓名 | 年級 | 班級 | 專項 | 家長姓名 | 家長聯絡方式 | 家長查詢碼 | 狀態 | 晚自習報名 |
|--------|------|------|------|------|----------|--------------|------------|------|------------|

- 狀態：在學 / 休學 / 畢業（只有「在學」會出現在點名）。
- 晚自習報名：例「一三五」，空白＝沒報名；晚自習時段只點有報名且當天的學生。

### 2. Attendance（每日點名紀錄）
| 日期 | 星期 | 時段 | 學生ID | 姓名 | 班級 | 專項 | 狀態 | 請假原因 | 遲到分鐘 | 備註 | 記錄者 | 建立時間 | 修改時間 |
|------|------|------|--------|------|------|------|------|----------|----------|------|--------|----------|----------|

- 時段：晨操 / 專長訓練 / 晚自習。狀態：出席 / 遲到 / 請假 / 未到。
- 同日同時段同學生重複點名會自動覆蓋（只留最新），並保留建立時間。

### 3. Users（帳號）
| 帳號 | 密碼 | 角色 | 專項 | 姓名 |
- 角色 `admin` 或 `coach`；專項 田徑 / 跆拳道 / 武術 / 全部。

### 4. 報表
按報表頁「匯出 Google Sheets」時自動覆寫產生。

## 六、出席率與紅黃綠燈
- 出席率 =（出席＋遲到）÷（出席＋遲到＋未到）；**請假不計入分母**。
- 🟢 ≥90%　🟡 80–89%　🔴 <80%　⚪ 無紀錄

## 七、需要關心學生（戰情室自動偵測）
1. 連續未到 2 次以上　2. 本週遲到 2 次以上　3. 本月請假 4 次以上
4. 今日未到未請假　5. 本月出席率 <80%（且當月已有 3 筆以上紀錄）
→ 可直接按「個別通知」產生溫和的家長訊息。

## 八、LINE 回報（三種）
1. **教練群**：日期/時段/應到/出席/遲到/請假/未到＋遲到、請假、未到名單。
2. **家長群**：語氣溫和、不公開點名個別學生，只報今日點名完成與提醒。
3. **個別家長**：未到或常遲到的學生，自動產生關心訊息。

## 九、現場操作小抄
**每天**：戰情室看今日狀況 → 點名頁（時段已依時間自動帶入）→ 用篩選找「尚未點名」→ 點四個大按鈕（可先「全部出席」再改特例）→ 儲存 → 按「教練群／家長群」複製貼 LINE。
